# v3 — Self-explaining diagnostics

Three layers that make failures *legible*, not just numeric — all surfaced in
the dashboard. This is the "AI evaluation with verification built in" part.

> Tag: `v3.0.0` · Phase P3 · builds on [v2](v2.md)

```bash
python -m eval.run --tag baseline            # now includes the diagnostics pass
python -m eval.run --tag quick --no-diagnostics   # skip it for a cheap run
```

---

## (a) Self-explaining failure drill-down

Every case below the faithfulness gate gets a **one-sentence judge reason**
naming the weakest-grounded claim (`eval/diagnostics.py::explain_failure`). It's
shown as a red callout in the dashboard drill-down, so a failing score comes
with a *why*, not just a number.

## (b) Cost-per-run tracking

Each run logs tokens + USD split across **generator / judge / diagnostics**, in
the result file and the dashboard sidebar:

```
cost (USD) 0.0663   (gen 0.0000 + judge 0.0607 + diag 0.0056)
```

The diagnostics layer (30 context audits + 10 failure explanations) adds only
**~$0.006** to a full run — cheap enough to run every time.

## (c) Context-correctness layer — the differentiator

An **independent judge audit** (`context_correctness`) asks a different question
than faithfulness: *do the retrieved passages actually contain what's needed for
the correct answer?* A RAG can be perfectly faithful to context that is stale or
wrong — and faithfulness alone will never catch it. When context is unsupported
**yet faithfulness is high**, the run raises a `stale_context` flag (⚠ in the
table, amber warning in the drill-down).

### It caught a real one

From the baseline run:

> **Q:** *"did Robert Wise have more award nominations than Zoltan Korda?"*
> **Answer:** "Yes, Robert Wise had more… nominated for Best Film Editing for
> *Citizen Kane*…" — confident and correct.
> **faithfulness = 0.75** (looks grounded) · **context recall = 0.00**
> **Auditor:** *"The retrieved passages do not provide any information about the
> number of award nominations for Robert Wise or Zoltan Korda."*

The model answered from its **parametric memory**, not the retrieved context —
a confident answer grounded in the *wrong* context. Faithfulness rated it 0.75;
the context-correctness layer flagged it. That gap is exactly the failure mode
this layer exists to surface.

**DoD:** self-explaining failure reasons, cost-per-run, and the
context-correctness flag all visible in the dashboard. ✅
